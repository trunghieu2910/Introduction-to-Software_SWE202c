KEY WORD AND QUIZZ

| Thuật ngữ (Tiếng Anh / Tiếng Việt) | Bản chất & Ý nghĩa |
| :---- | :---- |
| Generic Software (Phần mềm đại trà) | Phần mềm được tạo ra dựa trên nghiên cứu thị trường, bán cho số đông (VD: MS Word, Windows). Rủi ro về yêu cầu thấp nhưng cạnh tranh thị trường cao. |
| Custom Software (Phần mềm tùy chỉnh) | Phần mềm viết riêng theo đơn đặt hàng của một khách hàng cụ thể (VD: App quản lý kho cho nhà hàng X). Rủi ro cao vì dễ hiểu sai ý khách hàng, tốn nhiều công sức phân tích. |
| Embedded Software (Phần mềm nhúng) | Code nằm trong một thiết bị phần cứng (VD: Hệ điều hành lò vi sóng). Yêu cầu đã được chốt cứng. |
| Green field project (Dự án mới tinh) | Dự án làm lại từ đầu (từ con số 0), không phải kế thừa code cũ. |
| Evolutionary project (Dự án tiến hóa) | Dự án phổ biến nhất: Sửa lỗi, nâng cấp và thêm tính năng trên một hệ thống code có sẵn (Legacy system). |
| The Four P's (Nguyên tắc 4 chữ P) | 4 yếu tố PM phải quản lý: Problem (Vấn đề/Yêu cầu), People (Con người), Process (Quy trình), Product (Sản phẩm đầu ra). (Lưu ý: Không có chữ Plan). |
| Project Risk (Rủi ro dự án) | Bất kỳ điều gì có thể làm dự án chệch hướng. Được đo lường bằng Khả năng xảy ra (Likelihood) x Mức độ tác động (Impact). |
| Scope (Phạm vi dự án) | Định nghĩa rõ ràng "Ranh giới": Cái gì team sẽ làm và cái gì team sẽ KHÔNG làm. Đây là việc ĐẦU TIÊN phải chốt. |
| Milestone (Cột mốc) | Một điểm dừng mang tính quyết định của quản lý (management decision point) để đánh giá xem dự án có đang đi đúng hướng hay không. |
| Deliverable (Sản phẩm bàn giao) | Bất kỳ sản phẩm nào có thể chuyển giao được (VD: Bản thiết kế UML, một file code hoàn chỉnh, cuốn sách hướng dẫn). |
| Task vs. Activity | Task: Công việc siêu nhỏ giao cho 1 người.Activity: Một chùm các task liên quan nhau. |
| Master Schedule | Lịch trình tổng quan ở mức độ cao nhất (Macro) dùng để báo cáo cho cấp quản lý và khách hàng. |
