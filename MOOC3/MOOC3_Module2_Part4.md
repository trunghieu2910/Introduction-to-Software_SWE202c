QUIZ AND KEYWORD 

| Từ khóa (Tiếng Anh / Tiếng Việt) | Bản chất & Ý nghĩa |
| :---- | :---- |
| **Waterfall Model** *(Thác nước)* | Mô hình tuyến tính, nguyên khối (Monolithic). Mọi thứ làm 1 lần từ trên xuống, yêu cầu phải chốt cứng từ đầu. Tính kỷ luật (discipline) rất cao. |
| **Prototyping** *(Bản mẫu)* | Mô hình chuyên trị các dự án có yêu cầu "mơ hồ" (vague). Làm bản nháp cho khách hàng dùng thử để lấy phản hồi. |
| **Spiral Model** *(Xoắn ốc)* | Mô hình duy nhất nhấn mạnh vào khâu **Phân tích Rủi ro (Risk Analysis)** ở mỗi vòng lặp. |
| **Iterative** *(Lặp lại)* | Làm toàn bộ hệ thống ngay từ đầu nhưng ở dạng thô sơ (Full system, partial functionality), sau đó gọt giũa cho đẹp dần lên. |
| **Incremental** *(Tăng dần)* | Xây dựng từng phần của hệ thống một cách hoàn chỉnh (Partial system, full functionality) rồi ghép lại. |
| **Agile** *(Linh hoạt)* | Triết lý phát triển ưu tiên: Cá nhân tương tác, Phần mềm chạy được, Hợp tác khách hàng, và Phản hồi sự thay đổi (Responsiveness to change). |
| **Scrum** | Khung làm việc (Framework) phổ biến nhất của Agile, vận hành dựa trên các vòng lặp ngắn. |
| **Sprint** | Một vòng lặp ngắn (2-4 tuần) trong Scrum để tạo ra phần mềm chạy được. Không được đổi yêu cầu khi Sprint đang chạy. |
| **Product Owner (PO)** | Vai trò đại diện cho khách hàng, người quyết định tầm nhìn và độ ưu tiên của tính năng. |
| **Scrum Master** | Vai trò "Người bảo vệ team", giúp team gỡ rối, tổ chức họp hành và tuân thủ luật Scrum (giống PM). |
| **Product Backlog** | "Danh sách mong ước" chứa mọi tính năng cần làm của dự án. |
| **Burndown Chart** | Biểu đồ theo dõi tiến độ công việc **còn lại** (đường đi xuống dần về 0). |
| **Extreme Programming (XP)** | Lập trình cực hạn, nhấn mạnh việc **TDD** (viết test trước khi code) và **Pair Programming** (2 người code 1 máy). |
| **Continuous Integration (CI)** | Tích hợp liên tục, ngày nào cũng đẩy code lên kho chung để hệ thống tự động kiểm tra và ghép code. |
| **Unified Process (UP)** | Quy trình Hợp nhất, bao gồm 4 pha: **Inception** (Khởi tạo), **Elaboration** (Thiết kế kiến trúc), **Construction** (Xây dựng/Code), **Transition** (Chuyển giao). |
