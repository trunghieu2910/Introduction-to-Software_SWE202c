* **Defensive Programming (Lập trình phòng thủ):** Nghệ thuật viết code sao cho dù người dùng có nhập dữ liệu "ngu ngốc" đến mấy, hay hệ thống có rớt mạng, app của bạn vẫn sống nhăn răng chứ không bị văng lỗi (crash).

🐛 **Giai đoạn 2: Sửa chữa & Quản lý (Mô-đun 4\)**

* **Debugging:** Cách bắt con bọ (bug) đang lẩn trốn trong hàng ngàn dòng code.  
* **Configuration Management:** Cách quản lý các phiên bản code (như Git/GitHub), để team nhiều người cùng code mà không bị đè lên nhau.

🧪 **Giai đoạn 3: "Thánh địa" Kiểm thử \- Testing (Mô-đun 5, 6, 7\)**

Đây là linh hồn của khóa học này\!

* **White Box Testing (Kiểm thử Hộp trắng):** Bạn sẽ nhìn xuyên thấu vào mã nguồn (source code) để test từng vòng lặp for, từng câu lệnh if...else.  
* **Black Box Testing (Kiểm thử Hộp đen):** Nhắm mắt lại, coi phần mềm như một chiếc hộp đen. Bạn đóng vai người dùng, chỉ nhập Input và chờ xem Output có đúng như tài liệu Đặc tả (SRS) đã viết hay không.  
* **Acceptance Testing (Kiểm thử Nghiệm thu):** Trận chiến cuối cùng. Mang phần mềm đi demo cho khách hàng để họ gật đầu trả tiền.
