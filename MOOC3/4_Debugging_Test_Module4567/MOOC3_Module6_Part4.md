### **Bảng 1: Từ điển Thuật ngữ Kỹ thuật (Mô-đun 6\)**

| Từ khóa (Tiếng Anh / Tiếng Việt) | Bản chất & Ý nghĩa |
| :---- | :---- |
| **Condition Testing** *(Kiểm thử Điều kiện)* | Kỹ thuật kiểm thử Hộp trắng tập trung vào việc bẻ gãy các biểu thức logic (if/else, and/or) để tìm các lỗi sai toán tử hoặc sai logic. |
| **Loop Testing** *(Kiểm thử Vòng lặp)* | Kỹ thuật kiểm thử Hộp trắng chuyên trị các vòng lặp (for/while). Bắt buộc phải test các mốc: 0 lần, 1 lần, n-1 lần, n lần (tối đa), và n+1 lần (vượt rào). |
| **Data Flow Testing** *(Kiểm thử Luồng dữ liệu)* | Tập trung vào vòng đời của một biến: Từ lúc nó được khai báo/gán giá trị (Definition) cho đến lúc nó được mang ra tính toán (Use). |
| **Black Box Testing** *(Kiểm thử Hộp đen)* | Kiểm thử hệ thống mà không cần (hoặc không thể) nhìn thấy mã nguồn. Dựa hoàn toàn vào việc đối chiếu Input đầu vào và Output đầu ra so với tài liệu yêu cầu. |
| **Equivalence Partitioning** *(Phân vùng Tương đương)* | Kỹ thuật chia dữ liệu thành các nhóm (vùng) có tính chất giống nhau. Chỉ cần rút ra 1 giá trị đại diện ở mỗi vùng để test thay vì test toàn bộ. |
| **Boundary Value Analysis** *(Phân tích Giá trị Biên)* | Kỹ thuật tìm lỗi cực mạnh bằng cách nhắm thẳng vào các điểm ranh giới (điểm thấp nhất, cao nhất) và các giá trị nằm sát sạt ranh giới đó. |
| **Thread Testing** *(Kiểm thử Luồng sự kiện)* | Kiểm thử một chuỗi hành động liên hoàn liên quan đến nhiều đối tượng (Objects) hoặc nhiều Class tương tác với nhau. |
| **State-based Testing** *(Kiểm thử Trạng thái)* | Kiểm thử dựa trên việc ép phần mềm/đối tượng chuyển đổi qua các trạng thái khác nhau (VD: Từ 'Còn chỗ' sang 'Đầy') để xem hệ thống xử lý ra sao. |
| **Regression Testing** *(Kiểm thử Hồi quy)* | Việc lưu lại các Input đã từng gây ra lỗi (Bug) thành Test Case, và chạy lại chúng mỗi khi mã nguồn có sự thay đổi để đảm bảo lỗi cũ không quay lại. |
